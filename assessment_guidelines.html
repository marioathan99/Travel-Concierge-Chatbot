Vibe Coder Assessment

AI-Assisted Development with Cline

Assessment Overview

This assessment evaluates a candidate's ability to effectively collaborate with an AI assistant (Cline), manage development workflows, and build a functional prototype, with a focus on the "vibe" of their interaction and the quality of their output.

You will be tasked with building a simplified version of an AI-powered travel recommendation chatbot using Streamlit, integrating with the Claude API, and leveraging Model Context Protocol (MCP) servers. This exercise is designed to assess your technical skills, problem-solving approach, and your ability to work efficiently and effectively with an AI coding partner.

Key Evaluation Areas

  1. Cline Collaboration & Communication (40%): Clarity of prompts, problem decomposition, and overall synergy in the AI-assisted development process.
  2. Context Window Management (20%): Efficiency in managing conversation history and strategic use of information-gathering tools.
  3. Plan vs. Act Mode Mastery (20%): Strategic planning in "PLAN MODE" and effective execution in "ACT MODE".
  4. MCP Integration Skills (10%): Successful integration and creative application of MCP tools.
  5. Prototype Quality & Technical Excellence (10%): Functionality, code quality, and user experience of the final prototype.

Assessment Task: Simplified Travel Concierge Chatbot

Objective & Scenario

Build a Streamlit-based chatbot that can recommend travel destinations and activities, leveraging the Claude API for conversational intelligence and an MCP server for external data.

The chatbot should be able to:

  • Engage in a conversation about travel preferences.
  • Suggest destinations and activities using synthetic data.
  • Display recommendations in a clear format.
  • Utilize an MCP server to look up additional information.
  • Simulate a "booking" confirmation.

  • Language: Python 3.9+

Instructions

  1. Start in PLAN MODE to outline your approach with Cline.
  2. Switch to ACT MODE to implement your plan.
  3. Iterate with Cline for coding, debugging, and advice.
  4. Demonstrate at least one call to an MCP tool.
  5. Submit your `app.py`, `requirements.txt`, `README.md`, and Cline chat history.
